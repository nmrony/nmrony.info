Working Init script for Logstash Forwarder for CentOS 5/6

Nur Rony

Polyglot Programmer. Blindly love JS. An automation freak who love to cook code in React, Redux, RxJS and serve them using Docker. Married to the most beautiful girl.

Some days ago I am working one of a project where we need to install logstash-forwarder to collect logs from distributed servers. I found that the init script from here is not working well and after googling I found this script working well with CentOS 6

#!/bin/bash
# chkconfig: 345 80 20
# description: Logstash Forwarder
# processname: logstash-forwarder
# config: /etc/logstash-forwarder
# pidfile: /var/run/logstash-forwarder.pid

### BEGIN INIT INFO
# Provides: logstash-forwarder
# Required-Start: $local_fs $network $remote_fs
# Required-Stop: $local_fs $network $remote_fs
# Default-Start: 2 3 4 5
# Default-Stop: 0 1 6
# Short-Description: start and stop logstash-forwarder
# Description: Logstash Forwarder
### END INIT INFO

# Source function library.
. /etc/rc.d/init.d/functions

PATH=/sbin:/usr/sbin:/bin:/usr/bin

prog=logstash-forwarder  
DAEMON=/opt/logstash-forwarder/bin/logstash-forwarder  
pidfile=/var/run/$prog.pid  
lockfile=/var/lock/subsys/$prog

# load defaults

[ -e /etc/default/$prog ] &amp;&amp; . /etc/default/$prog
[ -e /etc/sysconfig/$prog ] &amp;&amp; . /etc/sysconfig/$prog

DAEMON_ARGS=&quot;${DAEMON_ARGS:--config /etc/logstash-forwarder -spool-size 100 -log-to-syslog}&quot;

start()  
{
    echo -n $&quot;Starting $prog: &quot;
    nohup $DAEMON $DAEMON_ARGS &lt;/dev/null &gt;/dev/null 2&gt;&amp;1 &amp;
    retval=$?
    pid=$!
    echo $pid &gt; $pidfile
    if [ rh_status_q ]; then
        touch $lockfile
        success
        echo
    fi
    return $retval
}

stop()  
{
    echo -n $&quot;Stopping $prog: &quot;
    killproc -p &quot;$pidfile&quot; $prog
    retval=$?
    [ -f &quot;$pidfile&quot; ] &amp;&amp; rm -f $pidfile
    echo
    [ $retval -eq 0 ] &amp;&amp; rm -f $lockfile
    return $retval
}

restart() {  
    stop
    start
}

reload() {  
    restart
}

force_reload() {  
    restart
}

rh_status() {  
    status -p $pidfile $prog
}

rh_status_q() {  
    rh_status &gt;/dev/null 2&gt;&amp;1
}

case &quot;$1&quot; in  
    start)
        rh_status_q &amp;&amp; exit 0
        $1
        ;;
    stop)
        rh_status_q || exit 0
        $1
        ;;
    restart)
        $1
        ;;
    reload)
        rh_status_q || exit 7
        $1
        ;;
    force-reload)
        force_reload
        ;;
    condrestart|try-restart)
        rh_status_q || exit 0
        restart
        ;;
    status)
        rh_status
        ;;
    *)
        echo &quot;Usage: $0 {start|stop|status|restart|condrestart|try-restart|reload|force-reload}&quot;
        exit 2
esac  
exit $?

Your instance also needed either of the following files 

/etc/defaults/logstash-forwarder
/etc/sysconfig/logstash-forwarder

The content of the file is 

DAEMON_ARGS='${DAEMON_ARGS:--config /etc/logstash-forwarder/logstash-forwarder.conf -spool-size 100}'
